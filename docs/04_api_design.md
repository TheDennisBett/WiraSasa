# API Design

## Phase 1
No backend integration exists yet. Repositories will be mocked until the service APIs are ready.

## Planned API Areas
- Auth: phone login, OTP verify, session refresh, social login handoff
- Discovery: categories, nearby providers, provider profile
- Requests: create request, schedule, accept/reject, status updates
- Tracking: provider coordinates, ETA updates, arrival events
- Billing: invoice creation, receipt retrieval
- Profile: mode switching, provider pricing, account settings
